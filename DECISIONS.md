## Clerk migration + Stripe (2026-07-16, user-directed)

13. **Clerk replaces Supabase Auth in Everdeck** (user confirmed after being shown it's a full identity migration). Done non-breaking: a schema migration adds `profiles.clerk_user_id`, drops the hard `auth.users` FKs, and routes RLS through `private.app_uid()` which resolves the current principal from `auth.jwt()->>'sub'` (Clerk) OR the legacy uuid (Supabase) — so the old prod kept working throughout. Verified via simulated JWT claims: Clerk identity provisions + is tenant-isolated; the demo Supabase user still resolves.

15. **Dedicated Clerk instance `alert-oarfish-38`** (user's choice) — the instance already registered as the Supabase third-party provider. The root Kairo app's `relevant-bat-62` keys do NOT work here; Everdeck needs alert-oarfish-38's own publishable + secret keys (pending from user). Edge verifier + env pinned to alert-oarfish-38.
16. **Provisioning via `ensure_workspace()` RPC** (idempotent, identity from verified JWT) called on first authenticated visit — replaces the `auth.users` signup trigger, which never fires for Clerk users. `sync_profile()` keeps email/name current; `toggle_star()` lets the client star without knowing its internal id.
